Estruturas Condicionais em JavaScript: Guia de Controle de Fluxo

Por padrão, os interpretadores de JavaScript executam o código de forma linear: de cima para baixo, linha por linha. As estruturas condicionais quebram essa linearidade. Elas atuam como bifurcações no código, permitindo que a aplicação tome decisões em tempo de execução com base em testes lógicos (expressões booleanas).

Se a condição estipulada for avaliada como verdadeira (true), um bloco específico de código é executado; se for falsa (false), o interpretador ignora esse bloco e segue outro caminho.

A Estrutura Base: if, else if e else

É a ferramenta mais comum para tomada de decisão no JavaScript. Ela avalia expressões lógicas e direciona o fluxo do script.

  • O Bloco if (Se)

    • Executa um bloco de código apenas se a condição entre parênteses for verdadeira.

    JavaScript

    const velocidade = 85;

    if (velocidade > 80) { console.log("Usuário multado por excesso de velocidade!"); }

  • O Bloco else (Senão)

    • Funciona como um plano de contingência. Ele executa um código alternativo caso a condição do if seja falsa.

    JavaScript

    const idade = 16;

    if (idade >= 18) { console.log("Acesso autorizado."); } else { console.log("Acesso negado. Usuário menor de idade."); }

  • O Encadeamento com else if (Senão Se)

    • Utilizado quando há mais de dois caminhos possíveis para resolver o problema. O JavaScript avalia as condições em ordem e executa o primeiro bloco que for verdadeiro.

    JavaScript

    const nota = 7.5;

    if (nota >= 9.0) { console.log("Excelente! Quadro de honra."); }

    else if (nota >= 7.0) { console.log("Aprovado regular."); } else if (nota >= 5.0) { console.log("Em recuperação."); }

    else { console.log("Reprovado."); }

Condição Múltipla: switch

Quando você precisa testar uma única variável contra múltiplos valores exatos fixos, o encadeamento de muitos else if pode tornar o código poluído e difícil de ler. O bloco switch organiza essa estrutura de forma limpa através de "casos".

Anatomia do switch:

  • case: Define o valor a ser comparado com a variável.

  • break: Interrompe a execução do switch. Se você esquecer o break, o interpretador continuará executando todos os próximos casos abaixo de forma cascata, mesmo que eles não correspondam ao valor (comportamento conhecido como fall-through).

  • default: Funciona como o else. É o código executado se nenhum dos casos anteriores for correspondido.

    JavaScript

    const diaSemana = 3;

    let nomeDia = "";

    switch (diaSemana) {

    case 1:
    nomeDia = "Segunda-feira"; break;

    case 2:
    nomeDia = "Terça-feira"; break;

    case 3:
    nomeDia = "Quarta-feira"; break;

    default:
    nomeDia = "Dia inválido ou final de semana"; }

    console.log(nomeDia); // Saída: Quarta-feira

O Operador Ternário (Condicional Curto)

O operador ternário é uma forma compacta de escrever um if / else que serve puramente para atribuir um valor ou retornar um dado. Ele recebe esse nome porque é o único operador do JavaScript que exige três operandos.

Sintaxe:

$$\text{condição } ? \text{ valor\_se\_verdadeiro} : \text{ valor\_se\_falso}$$ Exemplo Prático:

JavaScript

const saldo = 150;

// Utilizando IF / ELSE tradicional let statusConta;

if (saldo >= 0) { statusConta = "Positivo"; } else { statusConta = "Negativo"; }

// O mesmo comportamento refatorado com Operador Ternário:

const statusTernario = saldo >= 0 ? "Positivo" : "Negativo";
Nota de UI/UX e Código Limpo: Não encadeie múltiplos operadores ternários uns dentro dos outros (ex: condicao ? var1 : condicao2 ? var2 : var3). Isso destrói a legibilidade do código. Se houver mais de duas opções, prefira o if / else ou switch.

Conceito Avançado: Truthy e Falsy

No JavaScript, uma condição dentro do if não precisa ser necessariamente uma expressão matemática ou lógica pura (como x > 5). Você pode passar qualquer tipo de dado (strings, números, objetos) diretamente na condição.

O JavaScript converte esse dado implicitamente para um booleano (true ou false). É aqui que entram os conceitos de Truthy e Falsy:

Valores Falsy (Valores avaliados como false)

Existem exatamente 6 valores que o JavaScript sempre considerará falsos em uma condicional:

  1. false (O booleano nativo)
  2. 0 (O número zero, incluindo -0)
  3. "" ou '' (Strings vazias)
  4. null (Ausência intencional de valor)
  5. undefined (Variável declarada, mas não inicializada)
  6. NaN (Not a Number - erro matemático)

Valores Truthy (Valores avaliados como true)

Todo o restante do JavaScript é avaliado como verdadeiro. Isso inclui arrays vazios [], objetos vazios {} e strings com pelo menos um caractere de espaço " ".

Exemplo de uso prático em APIs:

JavaScript

const usuarioLogado = "Gabriel"; // String preenchida (Truthy)

if (usuarioLogado) { console.log(`Bem-vindo, ${usuarioLogado}!`); // Executa porque a string não está vazia }

else { console.log("Por favor, faça o login."); }

Curto-Circuito Lógico (Short-Circuit Evaluation)

Os operadores lógicos && (AND) e || (OR) podem ser usados fora dos blocos if para criar condicionais rápidas e elegantes. Eles interrompem a leitura assim que encontram o resultado definitivo da expressão.

Operador && (Executa se for verdadeiro)

Ele avalia o código da esquerda. Se for verdadeiro, ele avança e executa/retorna o código da direita. É muito utilizado em frameworks modernos (como o React) para renderizar elementos na tela.

JavaScript

const exibirMenu = true;

// Se exibirMenu for true, executa a função da direita. Se for false, o código para ali mesmo.

exibirMenu && renderizarMenuNavegacao();

Operador || (Garante um valor padrão se for falso)

Ele avalia o código da esquerda. Se for verdadeiro, ele para ali mesmo e o retorna. Se for falso (ou seja, se a variável vier nula ou vazia), ele avança e assume o valor de segurança à direita.

JavaScript

const nomeDigitado = ""; // Usuário não preencheu o campo (Falsy)

// Garante que o sistema tenha um nome válido para exibir

const nomeFinal = nomeDigitado || "Usuário Anônimo";

console.log(nomeFinal); // Saída: Usuário Anônimo
