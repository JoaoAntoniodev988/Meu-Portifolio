Variáveis

Uma variável é como uma caixa etiquetada onde você guarda uma informação para usar mais tarde. No JavaScript moderno, usamos duas palavras-chave principais para criar essas caixas:

const (Constante): Cria uma gaveta que, uma vez fechada com um valor dentro, nunca mais pode ser alterada. Use sempre esta por padrão.

let: Cria uma gaveta que pode ter o valor substituído ao longo do tempo. Use quando souber que o dado vai mudar (como a pontuação de um jogo).

(Nota: Você talvez veja o var em códigos antigos. Ele caiu em desuso porque causava alguns comportamentos confusos no código. Evite usar).

JavaScript

const nome = "Lucas"; // Este nome não muda

let pontos = 10; // O jogo começou, o jogador tem 10 pontos e pode mudar no decorrer do jogo.

Regras de identificadores das variáveis

Aborda as regras gramaticais para dar nome às coisas no JavaScript.Em programação, um Identificador é simplesmente o nome que você escolhe para suas variáveis, constantes, funções ou classes. O JavaScript possui regras rígidas sobre o que é permitido e o que vai quebrar o seu código. Exemplos práticos:

Podem começar com letra, $ ou _:
O primeiro caractere do nome da sua variável é o mais restrito. Ele só pode ser uma letra (maíuscula ou minúscula), o símbolo do cifrão ($) ou o underline (_).

JavaScript

const nome = "Lucas"; // ✅ Válido (começa com letra)

const $preco = 10.90; // ✅ Válido (muito usado em bibliotecas como jQuery ou RxJS)

const _id = "usr_123"; // ✅ Válido (comum para indicar variáveis privadas/internas)

Não podem começar com números:
Embora você possa usar números no nome da variável, eles nunca podem vir na primeira posição.

JavaScript

const 1ano = 2026; // ❌ Erro de Sintaxe! O JavaScript se confunde achando que é um número puro.

const ano1 = 2026; // ✅ Válido! O número no final ou meio é totalmente permitido.

É possível usar letras ou números:
Depois do primeiro caractere, você está livre para misturar letras (de A a Z) e números (de 0 a 9).

JavaScript

const apartamento202 = true; // ✅ Perfeito

É possível usar acentos e símbolos:
O JavaScript moderno aceita caracteres Unicode, o que significa que tecnicamente você pode usar acentos. Porém, há uma pegadinha de boa prática aqui!

JavaScript

const preço = 50; // ✅ Funciona no JavaScript, mas...

const calça = "Jeans"; // ✅ Também funciona…

⚠️ Alerta de Boa Prática: Embora o JavaScript permita acentos, a comunidade global de programação evita ao máximo. Usar acentos ou o ç pode gerar problemas de codificação quando seu código for rodar em servidores internacionais ou quando você trabalhar com programadores estrangeiros. Prefira preco e calca.

Não podem conter espaços
Uma variável precisa ser uma palavra única e contínua. O computador não entende que duas palavras separadas por espaço pertencem ao mesmo nome.

JavaScript

const nome usuario = "Mariana"; // ❌ Erro de Sintaxe! O espaço quebra o código.

Para resolver isso, a comunidade JavaScript usa um padrão chamado camelCase (onde a primeira palavra é minúscula e as seguintes começam com letra maiúscula, parecendo as corcovas de um camelo):

JavaScript

const nomeUsuario = "Mariana"; // ✅ Perfeito! Padrão camelCase.

Não podem ser palavras reservadas
O JavaScript tem uma lista de palavras que pertencem à própria linguagem (como comandos e instruções). Você não pode usar essas palavras como nome das suas variáveis, senão o motor do navegador entra em curto-circuito.

JavaScript

const let = 10; // ❌ Erro! "let" é uma palavra reservada para criar variáveis.

const function = 5; // ❌ Erro! "function" é reservada para criar funções.

const const = "Oi"; // ❌ Erro! "const" é reservada.

Tipos de Dados: O que vai dentro das variaveis

number (Número): é Qualquer valor numérico, seja positivo, negativo, inteiro ou decimal. Diferente de outras linguagens que separam números inteiros de números com vírgula, o JavaScript coloca tudo na mesma caixa chamada number. Exemplo:

JavaScript

const idade = 26; // Inteiro

const preco = 99.90; // Decimal (sempre use ponto, não vírgula)

const temperatura = -5; // Negativo

Infinity (Infinito): Ele aparece quando você faz uma operação matemática cujo resultado é maior do que o maior número que o JavaScript consegue processar, ou em divisões por zero. Este é um valor numérico especial que representa, literalmente, o infinito matemático. Exemplo:

JavaScript

const resultado = 10 / 0;

console.log(resultado); // Vai exibir: Infinity

NaN (Not a Number / Não é um Número): Ele serve para indicar que uma operação matemática falhou ou deu um resultado inválido/impossível. Apesar do nome significar "Não é um Número", o tipo dele para o JavaScript ainda é considerado um number especial! Se você tentar multiplicar um texto por um número. Exemplo:

JavaScript

const resultadoInvalido = "Olá" * 5;

console.log(resultadoInvalido); // Vai exibir: NaN

string (Texto): são Dados textuais que precisam estar envolvidos por aspas simples ('), duplas (") ou crases (` `). Como vimos antes, serve para qualquer cadeia de caracteres (letras, palavras, frases, códigos). Exemplo:

JavaScript

const nome = 'Ana';

const frase = "Estou aprendendo JS";

boolean (Booleano): É um tipo de dado que só aceita dois valores possíveis: true (verdadeiro) ou false (falso). O tipo de tomada do JavaScript: ou está ligado, ou está desligado. Exemplo:

JavaScript

const estaChovendo = false;

const maiorDeIdade = true;

null (Nulo): Usa-se o null quando quer dizer explicitamente para o computador: "Esta variável existe, mas ela está vazia de propósito neste momento". O null representa a ausência intencional de um valor. Exemplo: Um campo de "segundo nome" em um formulário que o usuário não preencheu.

JavaScript

let segundoNome = null; // O desenvolvedor definiu que está vazio

undefined (Indefinido): Ao contrário do null (que é intencional), o undefined é o padrão do JavaScript para variáveis que foram declaradas com let, mas não receberam nenhum valor inicial. O undefined significa que algo foi criado, mas nunca recebeu nenhum conteúdo.

JavaScript

let minhaVariavel;

console.log(minhaVariavel); // Vai exibir: undefined (porque não ganhou valor)

object (Objeto): É uma coleção de dados relacionados. É como uma ficha cadastral de alguma coisa do mundo real. Aqui entramos nos tipos estruturais. Um objeto permite agrupar várias informações em um único lugar usando pares de chave: valor.

JavaScript

  const carro = {
        marca: "Toyota",
        ano: 2024,
        ligado: true
    };

Array (Lista): É uma caixa onde você coloca vários dados em fila, separados por vírgulas e envolvidos por colchetes [ ]. O Array é, na verdade, um tipo especial de Objeto, mas que funciona como uma lista ordenada. Exemplo:

JavaScript

const frutas = ["Maçã", "Banana", "Morango"];

const numerosSorteados = [12, 45, 23, 8];

function (Função): É um "verbo" ou uma "máquina" que processa dados e te devolve uma resposta. Você pode guardá-las dentro de variáveis se quiser. No JavaScript, as funções também são tratadas como um tipo de dado. Elas são blocos de código que realizam uma ação quando são chamadas.

JavaScript

  function saudar() {
        return "Olá, mundo!";
    }
