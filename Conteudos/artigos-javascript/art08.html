Estruturas de Repetição em JavaScript: Automação e Loops

No desenvolvimento de software, frequentemente precisamos executar o mesmo bloco de código várias vezes seguidas (por exemplo, ler os 50 produtos de um carrinho, renderizar 10 posts na tela ou rodar um cálculo até atingir um limite).

Em vez de copiar e colar linhas de código, utilizamos estruturas de repetição (conhecidas como Loops ou Laços). Elas repetem uma instrução com base em uma condição lógica: enquanto a condição for verdadeira (true), o bloco continua rodando; no momento em que se torna falsa (false), o laço é quebrado e o programa segue em frente.

Aviso de Performance (Loop Infinito): Se a condição do seu laço de repetição nunca se tornar falsa, o código rodará infinitamente. Isso consome 100% do processamento disponível e trava a aba do navegador do usuário.

Estrutura Controlada por Contador: for

O laço for é a estrutura ideal quando você sabe exatamente quantas vezes precisa repetir aquela ação. Ele centraliza em uma única linha as três etapas essenciais de um loop: a inicialização, a condição de parada e o incremento.

Sintaxe:

JavaScript

for (inicialização; condição; incremento) { // Código que será repetido }

Exemplo Prático:

JavaScript

// Contando de 1 até 5 for (let i = 1; i <= 5; i++) { console.log(`Contagem: ${i}`); }

  • let i = 1: Cria uma variável de controle (ponteiro) que começa no número 1.
  • i <= 5: A condição de teste. O loop vai rodar enquanto i for menor ou igual a 5.
  • i++: O incremento. Soma +1 na variável i ao final de cada repetição.

Estruturas Controladas por Condição: while e do...while

Essas estruturas são perfeitas para cenários onde você não sabe previamente quantas repetições serão necessárias, pois dependem de uma condição que muda dinamicamente durante o processo (como esperar o usuário acertar uma senha).

  • O Laço while (Enquanto): Ele avalia a condição antes de executar o bloco de código. Se a condição começar como falsa, o bloco dentro dele nunca será executado.

    let energia = 3;

Loops Especializados para Coleções: for...of e for...in

O JavaScript moderno oferece atalhos nativos elegantes para percorrer estruturas de dados compostas (como listas/arrays e objetos) sem a necessidade de gerenciar contadores manuais.

A. O Laço for...of (Iterando sobre Arrays): Lê e entrega o valor de cada elemento de uma lista, um por um, do início ao fim.

JavaScript

const tecnologias = ["HTML", "CSS", "JavaScript", "React"];

for (const tech of tecnologias) { console.log(`Estudando a tecnologia: ${tech}`); }

// Saída: Entrega os nomes das tecnologias diretamente.

B. O Laço for...in (Iterando sobre Objetos): Diferente do anterior, ele é projetado para percorrer as propriedades (chaves) de um objeto literal.

JavaScript

const usuario = { nome: "Alice", idade: 25, profissao: "Engenheira de Software" };

for (const chave in usuario) { console.log(`${chave}: ${usuario[chave]}`); }

/* Saída: nome: Alice idade: 25 profissao: Engenheira de Software */

Comandos de Interrupção de Fluxo: break e continue

Às vezes, precisamos interferir no comportamento padrão do loop antes que a condição principal seja atingida. Para isso, o JavaScript disponibiliza duas palavras-chave:

A. O Comando break: Interrompe e encerra o loop imediatamente, jogando o fluxo do programa para a linha logo após a estrutura de repetição.

JavaScript

const listaNumeros = [10, 20, 35, 40, 50];

for (const num of listaNumeros) { if (num === 35) {

console.log("Número 35 encontrado! Parando a busca."); break; // O loop é cancelado e ignora os números 40 e 50.

}

console.log(`Analisando: ${num}`);

}

B. O Comando continue: Não encerra o loop. Em vez disso, ele pula apenas a repetição atual e vai direto para o próximo passo do laço.

JavaScript

// Imprimindo apenas números ímpares (pulando os pares)

for (let i = 1; i <= 5; i++) { if (i % 2 === 0) {

continue; // Se for par, ignora o console.log abaixo e vai para o próximo número

}

console.log(`Número Ímpar: ${i}`);

}
