A diferencia entre javascript e EcmaScript

Para entender a diferença de forma bem simples, pense na relação entre o Português (a língua) e o Dicionário da Academia de Letras (as regras).

ECMAScript é a regra (o padrão): É um documento oficial que dita como a linguagem deve funcionar. Ele diz quais palavras-chave existem, como os loops devem se comportar e quais são os tipos de dados. Ele existe apenas no papel.

JavaScript é a língua (a implementação): É a linguagem real que você escreve no seu editor de código e que o navegador roda. O JavaScript segue as regras que o ECMAScript determinou.

Uma Breve História (Para fazer sentido)

Nos anos 90, o JavaScript foi criado para o navegador Netscape.Logo depois, a Microsoft criou a sua própria versão para o Internet Explorer, chamada JScript.

Como cada navegador começou a criar suas próprias regras, a internet virou uma bagunça (um código que funcionava em um navegador quebrava no outro).

Para resolver isso, eles pediram para uma organização internacional chamada ECMA padronizar a linguagem. Nasceu aí o ECMAScript.

Por que você ouve falar tanto de "ES6", "ES7", etc.?

ES é a abreviação de EcmaScript. O número ao lado é a versão da regra.

ES5 (2009): É o JavaScript mais antigo.

ES6 / ES2015: Foi a maior atualização da história da linguagem. Ela mudou o JavaScript de patamar, trazendo coisas que usamos hoje como o let, const, as Arrow Functions e as Template Strings (as crases para textos).

Atualizações Anuais: Hoje em dia, o comitê do ECMAScript lança uma versão nova todo ano (ES2024, ES2025, etc.) trazendo pequenas melhorias.

💡 Resumo da ópera: Você não precisa aprender duas coisas diferentes. Quando você estuda JavaScript moderno, você já está estudando as versões mais recentes do ECMAScript.

Variáveis

Uma variável é como uma caixa etiquetada onde você guarda uma informação para usar mais tarde. No JavaScript moderno, usamos duas palavras-chave principais para criar essas caixas:

const (Constante): Cria uma gaveta que, uma vez fechada com um valor dentro, nunca mais pode ser alterada. Use sempre esta por padrão.

let: Cria uma gaveta que pode ter o valor substituído ao longo do tempo. Use quando souber que o dado vai mudar (como a pontuação de um jogo).

(Nota: Você talvez veja o var em códigos antigos. Ele caiu em desuso porque causava alguns comportamentos confusos no código. Evite usar).

JavaScript

const nome = "Lucas"; // Este nome não muda

let pontos = 10; // O jogo começou, o jogador tem 10 pontos e pode mudar no decorrer do jogo.

Regras de identificadores das variáveis

Aborda as regras gramaticais para dar nome às coisas no JavaScript.Em programação, um Identificador é simplesmente o nome que você escolhe para suas variáveis, constantes, funções ou classes. O JavaScript possui regras rígidas sobre o que é permitido e o que vai quebrar o seu código. Exemplos práticos:

Podem começar com letra, $ ou _:
O primeiro caractere do nome da sua variável é o mais restrito. Ele só pode ser uma letra (maíuscula ou minúscula), o símbolo do cifrão ($) ou o underline (_).

JavaScript

const nome = "Lucas"; // ✅ Válido (começa com letra)

const $preco = 10.90; // ✅ Válido (muito usado em bibliotecas como jQuery ou RxJS)

const _id = "usr_123"; // ✅ Válido (comum para indicar variáveis privadas/internas)

Não podem começar com números:
Embora você possa usar números no nome da variável, eles nunca podem vir na primeira posição.

JavaScript

const 1ano = 2026; // ❌ Erro de Sintaxe! O JavaScript se confunde achando que é um número puro.

const ano1 = 2026; // ✅ Válido! O número no final ou meio é totalmente permitido.

É possível usar letras ou números:
Depois do primeiro caractere, você está livre para misturar letras (de A a Z) e números (de 0 a 9).

JavaScript

const apartamento202 = true; // ✅ Perfeito

É possível usar acentos e símbolos:
O JavaScript moderno aceita caracteres Unicode, o que significa que tecnicamente você pode usar acentos. Porém, há uma pegadinha de boa prática aqui!

JavaScript

const preço = 50; // ✅ Funciona no JavaScript, mas...

const calça = "Jeans"; // ✅ Também funciona…

⚠️ Alerta de Boa Prática: Embora o JavaScript permita acentos, a comunidade global de programação evita ao máximo. Usar acentos ou o ç pode gerar problemas de codificação quando seu código for rodar em servidores internacionais ou quando você trabalhar com programadores estrangeiros. Prefira preco e calca.

Não podem conter espaços
Uma variável precisa ser uma palavra única e contínua. O computador não entende que duas palavras separadas por espaço pertencem ao mesmo nome.

JavaScript

const nome usuario = "Mariana"; // ❌ Erro de Sintaxe! O espaço quebra o código.

Para resolver isso, a comunidade JavaScript usa um padrão chamado camelCase (onde a primeira palavra é minúscula e as seguintes começam com letra maiúscula, parecendo as corcovas de um camelo):

JavaScript

const nomeUsuario = "Mariana"; // ✅ Perfeito! Padrão camelCase.

Não podem ser palavras reservadas
O JavaScript tem uma lista de palavras que pertencem à própria linguagem (como comandos e instruções). Você não pode usar essas palavras como nome das suas variáveis, senão o motor do navegador entra em curto-circuito.

JavaScript

const let = 10; // ❌ Erro! "let" é uma palavra reservada para criar variáveis.

const function = 5; // ❌ Erro! "function" é reservada para criar funções.

const const = "Oi"; // ❌ Erro! "const" é reservada.

Tipos de Dados: O que vai dentro das variaveis

number (Número): é Qualquer valor numérico, seja positivo, negativo, inteiro ou decimal. Diferente de outras linguagens que separam números inteiros de números com vírgula, o JavaScript coloca tudo na mesma caixa chamada number.Exemplo:

JavaScript

const idade = 26; // Inteiro

const preco = 99.90; // Decimal (sempre use ponto, não vírgula)

const temperatura = -5; // Negativo

Infinity (Infinito): Ele aparece quando você faz uma operação matemática cujo resultado é maior do que o maior número que o JavaScript consegue processar, ou em divisões por zero.Este é um valor numérico especial que representa, literalmente, o infinito matemático. Exemplo:

JavaScript

const resultado = 10 / 0;

console.log(resultado); // Vai exibir: Infinity

NaN (Not a Number / Não é um Número): Ele serve para indicar que uma operação matemática falhou ou deu um resultado inválido/impossível. Apesar do nome significar "Não é um Número", o tipo dele para o JavaScript ainda é considerado um number especial! Se você tentar multiplicar um texto por um número. Exemplo:

JavaScript

const resultadoInvalido = "Olá" * 5;

console.log(resultadoInvalido); // Vai exibir: NaN

string (Texto): são Dados textuais que precisam estar envolvidos por aspas simples ('), duplas (") ou crases (` `). Como vimos antes, serve para qualquer cadeia de caracteres (letras, palavras, frases, códigos). Exemplo:

JavaScript

const nome = 'Ana';

const frase = "Estou aprendendo JS";

boolean (Booleano): Um tipo de dado que só aceita dois valores possíveis: true (verdadeiro) ou false (falso). O tipo de tomada do JavaScript: ou está ligado, ou está desligado. Exemplo:

JavaScript

const estaChovendo = false;

const maiorDeIdade = true;

null (Nulo): Você usa o null quando quer dizer explicitamente para o computador: "Esta variável existe, mas ela está vazia de propósito neste momento". O null representa a ausência intencional de um valor. Exemplo: Um campo de "segundo nome" em um formulário que o usuário não preencheu.

JavaScript

let segundoNome = null; // O desenvolvedor definiu que está vazio

undefined (Indefinido): Ao contrário do null (que é intencional), o undefined é o padrão do JavaScript para variáveis que foram declaradas com let, mas não receberam nenhum valor inicial. O undefined significa que algo foi criado, mas nunca recebeu nenhum conteúdo. Exemplo:

JavaScript

let minhaVariavel;

console.log(minhaVariavel); // Vai exibir: undefined (porque não ganhou valor)

object (Objeto): Uma coleção de dados relacionados. É como uma ficha cadastral de alguma coisa do mundo real. Aqui entramos nos tipos estruturais. Um objeto permite agrupar várias informações em um único lugar usando pares de chave: valor. Exemplo:

JavaScript

  const carro = {
        marca: "Toyota",
        ano: 2024,
        ligado: true
    };

Array (Lista): Uma caixa onde você coloca vários dados em fila, separados por vírgulas e envolvidos por colchetes [ ]. O Array é, na verdade, um tipo especial de Objeto, mas que funciona como uma lista ordenada. Exemplo:

JavaScript

const frutas = ["Maçã", "Banana", "Morango"];

const numerosSorteados = [12, 45, 23, 8];

function (Função): Um "verbo" ou uma "máquina" que processa dados e te devolve uma resposta. Você pode guardá-las dentro de variáveis se quiser. No JavaScript, as funções também são tratadas como um tipo de dado. Elas são blocos de código que realizam uma ação quando são chamadas. Exemplo:

JavaScript

  function saudar() {
        return "Olá, mundo!";
    }

Conversão de Dados (Type Casting / Coercion)

Às vezes, você recebe um dado em um formato (como um texto vindo de um formulário), mas precisa dele em outro formato (como um número para fazer uma conta). No JavaScript, existem duas formas de conversão:

Conversão Explícita (Você força a mudança)

É quando você usa comandos da linguagem para transformar o dado conscientemente. Para Número: Usa-se Number(), parseInt() (para inteiros) ou parseFloat() (para decimais). Para Texto: Usa-se String() ou o método .toString().

JavaScript

const textoIdade = "25";

const idadeNumero = Number(textoIdade); // Transforma em número real: 25

const numero = 100;

const numeroTexto = String(numero); // Transforma em texto: "100"

Conversão Implícita (O JavaScript tenta adivinhar)

O JavaScript é uma linguagem "compreensiva" até demais. Se você tentar somar um número com um texto, em vez de dar erro, ele converte o número em texto e junta os dois.

JavaScript

const resultado = "5" + 2;

console.log(resultado); // Vai exibir: "52" (Texto), e não 7!

Manipulação de Dados

Manipular dados significa alterar o seu estado ou extrair informações úteis deles. Cada tipo de dado tem seus próprios "superpoderes" (métodos).

Manipulando Strings (Textos)

  • Descobrir o tamanho (.length): Conta quantos caracteres o texto tem.
  • Mudar a caixa (.toUpperCase() / .toLowerCase()): Deixar tudo em maiúsculo ou minúsculo.
  • Cortar pedaços (.slice()): Extrair apenas uma parte do texto.

JavaScript

const frase = "JavaScript é incrível!";

console.log(frase.length); // 22 (contando os espaços)

console.log(frase.toUpperCase()); // "JAVASCRIPT É INCRÍVEL!"

Manipulando Números (Objeto Math)

Para contas complexas ou ajustes, usamos o objeto nativo Math.

  • Arredondar: Math.round() (arredonda para o mais próximo)
  • Math.ceil() (para cima) ou Math.floor() (para baixo).
  • Gerar número aleatório: Math.random().

JavaScript

const nota = 7.6;

console.log(Math.round(nota)); // 8

console.log(Math.floor(nota)); // 7

Formatação de Dados

Formatar é preparar o texto para que ele seja exibido de forma bonita e legível para o usuário final.

Formatação de String

  • O Jeito Antigo: Concatenação. Juntar textos e variáveis usando o sinal de +. Costuma ficar confuso muito rápido.

    JavaScript

    const nome = "Carlos";

    const tecnologia = "React";

    console.log("O aluno " + nome + " está estudando " + tecnologia + ".");

  • O Jeito Moderno: Template Strings (ES6). Usamos as crases (` `) e colocamos as variáveis dentro de ${}. O código fica muito mais limpo e aceita quebra de linhas.

    JavaScript

    const nome = "Carlos";

    const tecnologia = "React"; // Muito mais fácil de ler e manter!

    console.log(`O aluno ${nome} está estudando ${tecnologia}.`);

Formatação de Números

Mostrar números brutos no sistema pode parecer pouco profissional (ex: 1250.5). Precisamos formatá-los para a realidade do usuário, como moedas ou casas decimais fixas.

  • Fixar Casas Decimais (.toFixed()): Determina quantas casas vão aparecer após o ponto. Atenção: este método transforma o número em uma string.

    JavaScript

    const precoBruto = 19.99321;

    console.log(precoBruto.toFixed(2)); // "19.99"

  • Formatação de Moedas e Regiões (Intl.NumberFormat): A forma mais profissional e poderosa do JavaScript para formatar moedas, porcentagens e padrões de países automaticamente.

    JavaScript

    const salario = 5000.50; // Formatando para o Real Brasileiro (R$)

    const formatadorReal = new Intl.NumberFormat('pt-BR', { style: 'currency', currency: 'BRL' });

    console.log(formatadorReal.format(salario)); // Exibe: R$ 5.000,50

Esses quatro pontos dão a você o controle total sobre as informações que entram e saem do seu programa.

Operadores

  • Operadores Aritméticos: São os operadores matemáticos clássicos que usamos para fazer cálculos.

    Os Símbolos:

    • + (Adição) / - (Subtração)
    • * (Multiplicação) / / (Divisão)
    • % (Módulo / Resto da divisão): Muito usado para descobrir se um número é par ou ímpar.
    • ** (Exponenciação): Elevar um número a uma potência.
    Exemplo:

    JavaScript

    const resto = 11 % 2; // O resultado é 1 (porque 11 dividido por 2 dá 5 e sobra 1)

    const quadrado = 3 ** 2; // 3 elevado a 2 = 9

  • Operadores Relacionais (Comparação): Servem para comparar dois valores e avaliar a relação entre eles. O resultado dessa comparação é sempre um booleano (true ou false).

    Os Símbolos:

    • > (Maior que) / < (Menor que)
    • >= (Maior ou igual) / <= (Menor ou igual)
    • == (Igualdade ampla): Compara apenas o valor, ignorando o tipo de dado. (Evite usar)
    • === (Igualdade estrita): Compara o valor E o tipo de dado. (Sempre use este)
    • !== (Diferente estrito): Verifica se os valores ou tipos são diferentes.

    Exemplo:

    JavaScript

    console.log(5 == "5"); // true (o JS converte o texto para número antes de comparar)

    console.log(5 === "5"); // false (são iguais no valor, mas um é Number e o outro é String)
  • Operadores Lógicos: Servem para combinar várias comparações relacionais e criar regras de negócio mais complexas.

    Os Símbolos:

    • && (E / AND): A frase inteira só é verdade se tudo for verdade.
    • || (OU / OR): A frase inteira é verdade se pelo menos um elemento for verdade.
    • ! (NÃO / NOT): Inverte o booleano. Se algo era true, vira false.

    Exemplo:

    JavaScript

    const temIdade = true;

    const temPermissaoDosPais = false;

    // O jovem pode viajar se tiver idade OU a permissão

    const podeViajar = temIdade || temPermissaoDosPais; // true

  • Operador Ternário: O operador ternário é um atalho elegante para criar uma estrutura de decisão rápida (um if / else de uma linha só). Ele recebe esse nome porque é o único operador do JavaScript que usa três argumentos.

    A Estrutura: condição ? valor_se_verdade : valor_se_falso;

    Exemplo:

    JavaScript

    const idade = 20;

    // Se idade for maior ou igual a 18, retorna "Maior de idade", senão retorna "Menor de idade"

    const status = idade >= 18 ? "Maior de idade" : "Menor de idade";

    console.log(status); // "Maior de idade"

     O operador ternário e os operadores relacionais (===) são os que os iniciantes mais demoram para se acostumar, mas são os mais usados no dia a dia.
  • Operadores de Atribuição: Servem para guardar ou atualizar o valor de uma variável. O operador básico você já conhece: o sinal de igual (=). Mas existem atalhos inteligentes:

    Os Símbolos:

    • = (Atribuição simples)
    • += (Soma e atribui): Pega o valor atual e soma mais um tanto.
    • -= (Subtrai e atribui)
    • *= (Multiplica e atribui)

    Exemplo:

    JavaScript

    let pontos = 10;

    // O jeito longo: pontos = pontos + 5;

    // O jeito inteligente (Operador de Atribuição):

    pontos += 5; // Agora pontos vale 15

  • Operadores de Atribuição Composta: Operadores de Atribuição Composta (ou simplesmente "simplificação de atribuição"). Em programação, passamos muito tempo atualizando o valor de uma variável com base no valor que ela já possuía antes. O JavaScript nos oferece esse "atalho" para não termos que repetir o nome da variável duas vezes.

    Assumindo que começamos com let n = 3:

    1. Adição Simplificada (+=)
      • Forma longa: n = n + 4 (Pega o valor atual de n, que é 3, soma 4 e guarda o resultado 7 de volta em n).
      • Forma simplificada: n += 4
      • Resultado: 7
    2. Subtração Simplificada (-=)
      • Forma longa: n = n - 5 (Pega o valor atual de n, subtrai 5 e guarda o resultado).
      • Forma simplificada: n -= 5
      • Resultado: Se n era 7, 7 - 5 = 2.
    3. Multiplicação Simplificada (*=)
      • Forma longa: n = n * 4 (Multiplica o valor atual por 4).
      • Forma simplificada: n *= 4
      • Resultado: Se n era 2, 2 * 4 = 8.
    4. Divisão Simplificada (/=)
      • Forma longa: n = n / 2 (Divide o valor atual por 2).
      • Forma simplificada: n /= 2
      • Resultado: Se n era 8, 8 / 2 = 4.
    5. Exponenciação Simplificada (=)
      • Forma longa: n = n 2 (Eleva o valor de n ao quadrado).
      • Forma simplificada: n = 2
      • Resultado: Se n era 4, 4 2 (4 elevado a 2) = 16.
    6. Módulo Simplificado (%=)
      • Forma longa: n = n % 5 (Pega o resto da divisão de n por 5).
      • Forma simplificada: n %= 5
      • Resultado: Se n era 16, ao dividir 16 por 5 dá 3 (que dá 15) e resta 1. Logo, o resultado é 1.
    💡 Por que os programadores usam isso? No dia a dia, você verá quase todo mundo usando a versão da direita. Ela deixa o código mais limpo, mais rápido de ler e evita erros de digitação quando as variáveis possuem nomes longos, como:

    JavaScript

    // Ruim / Longo

    quantidadeItensNoCarrinho = quantidadeItensNoCarrinho + 1;

    // Lindo / Simplificado quantidadeItensNoCarrinho += 1;

Ordem de Precedência dos Operadores.

Ordem de Precedência dos Operadores. Assim como na matemática da escola nós aprendemos que a multiplicação deve ser feita antes da soma, o JavaScript também tem uma lista de prioridades para decidir o que ele vai calcular primeiro quando encontra uma linha de código cheia de operadores misturados. Vamos entender esse fluxo de prioridades dividido em grupos:

1º Grupo:

  • () Parênteses: Têm prioridade máxima absoluta sobre qualquer outra coisa. Se você quiser que uma soma seja feita antes de uma multiplicação, coloque-a entre parênteses.

  • Exponenciação: É resolvida logo em seguida.

  • / , * , %: Divisão, multiplicação e resto têm a mesma prioridade entre si (se aparecerem juntos, o JS resolve quem estiver mais à esquerda primeiro). Nota: O + e o - vêm logo abaixo deles.

2º Grupo:

Operadores Relacionais (Comparações) Depois que todas as contas matemáticas da linha foram resolvidas e viraram números limpos, o JavaScript passa para as comparações.

> , < , >= , <= , === , !==: Ele avalia se um número é maior, menor ou igual ao outro. O resultado desse grupo sempre se transforma em true ou false.

3º Grupo:

Operadores Lógicos (Decisões) Por último, o JavaScript pega os resultados true ou false das comparações e aplica as regras lógicas. Eles possuem uma ordem interna muito rígida entre si: ! (NÃO / NOT): É o primeiro a agir, invertendo o valor lógico que estiver colado nele.

&& (E / AND): É o próximo a ser resolvido.

|| (OU / OR): É o último da fila. Tem a menor prioridade de todos.

Verificando na Prática (O poder dos parênteses) Veja como a ordem de precedência muda completamente o resultado final com os mesmos elementos:

Cenário A (Sem parênteses)

JavaScript

const resultado = 5 + 10 * 2 > 20 && true;

Aritmética primeiro: O JS faz 10 * 2 (20) e depois soma 5 $\rightarrow$ 25.

Relacional depois: Ele compara 25 > 20 $\rightarrow$ true.

Lógico por último: Ele avalia true && true $\rightarrow$ true.

Cenário B (Forçando com parênteses)

JavaScript

const resultado = (5 + 10) * 2 > 40 || true;

Parênteses no topo: O JS ignora a regra da multiplicação e faz primeiro 5 + 10 (15).

Resto da aritmética: Faz 15 * 2 $\rightarrow$ 30.

Relacional: Compara 30 > 40 $\rightarrow$ false.

Lógico por último: Avalia false || true $\rightarrow$ true.

DOM

O DOM (Document Object Model) é o ponto de virada na jornada de qualquer pessoa que estuda JavaScript. É através dele que o JavaScript deixa de ser apenas lógica no console e passa a dar vida, cor e movimento às páginas web reais. Aqui está a estrutura completa para entender o DOM do zero:

O que é o DOM (Document Object Model)?

Quando você abre um site, o navegador lê o arquivo HTML e cria, na memória dele, uma representação viva e organizada daquela página. Essa estrutura é o DOM.

  • O Conceito: O DOM transforma o seu código HTML em um "objeto" gigante que o JavaScript consegue ler, alterar, adicionar ou apagar.

  • A Árvore do DOM (DOM Tree): O DOM organiza a página como se fosse uma árvore genealógica invertida. O topo de tudo é o document. Abaixo dele vêm a "tag pai" html que se ramifica nos "filhos" head e body , que por sua vez têm seus próprios filhos h1, p, a, button,.

Selecionando Elementos (Como o JS encontra as coisas)

Antes de mudar qualquer elemento na página, o JavaScript precisa "encontrar" esse elemento. Antigamente existiam muitos comandos para isso, mas hoje usamos principalmente dois:

  • document.querySelector(): É o canivete suíço. Ele encontra o primeiro elemento que corresponda ao seletor CSS que você passar (seja o nome da tag, uma classe ou um ID).

  • document.querySelectorAll(): Encontra todos os elementos que batem com a descrição e os coloca em uma lista (uma espécie de array).

    • JavaScript

    // Selecionando pelo nome da tag

    const titulo = document.querySelector("h1");

    // Selecionando por uma classe (usa o ponto .)

    const botaoSucesso = document.querySelector(".btn-enviar");

    // Selecionando por um ID (usa a hashtag #)

    const inputNome = document.querySelector("#campo-nome");

Manipulando Elementos (Como o JS altera a página)

Depois de selecionar o elemento e guardá-lo em uma variável, você pode usar as propriedades do JavaScript para modificar a página em tempo real.

  • Alterando Textos e Conteúdos

    • .innerText ou .textContent: Altera apenas o texto puro dentro do elemento.

    • .innerHTML: Permite alterar o texto e ainda injetar novas tags HTML lá dentro.

    • JavaScript

    titulo.innerText = "Bem-vindo ao meu site interativo!";
  • Alterando Estilos (CSS)

    Você pode mudar qualquer propriedade visual usando o atributo .style.

    JavaScript

    // Deixando o fundo do botão vermelho

    botaoSucesso.style.backgroundColor = "red";

    botaoSucesso.style.color = "white";

  • Manipulando Classes (A melhor prática)

    Em vez de mudar o CSS linha por linha no JavaScript, o ideal é criar uma classe no CSS e apenas "ligar ou desligar" essa classe usando o .classList.

    JavaScript

    titulo.classList.add("destaque"); // Adiciona a classe

    titulo.classList.remove("escondido"); // Remove a classe

    titulo.classList.toggle("modo-escuro"); // Se tiver a classe remove, se não tiver adiciona!

Eventos (Como o JS reage ao Usuário)

Os eventos são as ações que acontecem no navegador: um clique, um movimento de mouse, o pressionar de uma tecla ou o envio de um formulário. O JavaScript fica "escutando" esses eventos para disparar um bloco de código (função) quando eles acontecem.

Usamos o método addEventListener(), que precisa de dois argumentos: o nome do evento que ele deve ouvir e o que ele deve fazer quando o evento acontecer.

JavaScript

const botao = document.querySelector("#meu-botao");

// Criando a função de ação

 function darBoasVindas() {
        alert("Você clicou no botão! Parabéns!");
    }

🚀 Resumo do Fluxo do DOM

Toda e qualquer interatividade que você vê na internet hoje segue este mesmo ciclo de 3 passos:
  1. Selecionar: O JS encontra o elemento (ex: o botão do carrinho de compras).
  2. Escutar: O JS espera a ação do usuário (ex: o clique no botão).
  3. Executar: O JS roda uma função que altera a página (ex: adiciona "+1" no ícone do carrinho).

Arvore DOM

Para entender a Árvore DOM (DOM Tree), imagine que o navegador é um tradutor. Ele pega o arquivo de texto bruto (o seu código HTML) e o transforma em um grande mapa ramificado cheio de conexões.

Esse mapa tem o formato de uma árvore genealógica invertida, onde tudo nasce de uma única raiz e vai se dividindo em galhos (ramos) e folhas. Cada elemento, texto ou atributo do seu HTML se transforma em um Nó (Node) dentro dessa árvore. Veja como o navegador organiza essa hierarquia:

1. A Raiz (document) É a base de tudo. O objeto document é o ponto de partida para qualquer coisa que você queira fazer no JavaScript.
