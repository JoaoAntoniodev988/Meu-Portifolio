A diferencia entre javascript e EcmaScript

Para entender a diferença de forma bem simples, pense na relação entre o Português (a língua) e o Dicionário da Academia de Letras (as regras).

ECMAScript é a regra (o padrão): É um documento oficial que dita como a linguagem deve funcionar. Ele diz quais palavras-chave existem, como os loops devem se comportar e quais são os tipos de dados. Ele existe apenas no papel.

JavaScript é a língua (a implementação): É a linguagem real que você escreve no seu editor de código e que o navegador roda. O JavaScript segue as regras que o ECMAScript determinou.

Uma Breve História (Para fazer sentido)

Nos anos 90, o JavaScript foi criado para o navegador Netscape.Logo depois, a Microsoft criou a sua própria versão para o Internet Explorer, chamada JScript.

Como cada navegador começou a criar suas próprias regras, a internet virou uma bagunça (um código que funcionava em um navegador quebrava no outro).

Para resolver isso, eles pediram para uma organização internacional chamada ECMA padronizar a linguagem. Nasceu aí o ECMAScript.

Por que você ouve falar tanto de "ES6", "ES7", etc.?

ES é a abreviação de EcmaScript. O número ao lado é a versão da regra.

ES5 (2009): É o JavaScript mais antigo.

ES6 / ES2015: Foi a maior atualização da história da linguagem. Ela mudou o JavaScript de patamar, trazendo coisas que usamos hoje como o let, const, as Arrow Functions e as Template Strings (as crases para textos).

Atualizações Anuais: Hoje em dia, o comitê do ECMAScript lança uma versão nova todo ano (ES2024, ES2025, etc.) trazendo pequenas melhorias.

💡 Resumo da ópera: Você não precisa aprender duas coisas diferentes. Quando você estuda JavaScript moderno, você já está estudando as versões mais recentes do ECMAScript.

Variáveis

Uma variável é como uma caixa etiquetada onde você guarda uma informação para usar mais tarde. No JavaScript moderno, usamos duas palavras-chave principais para criar essas caixas:

const (Constante): Cria uma gaveta que, uma vez fechada com um valor dentro, nunca mais pode ser alterada. Use sempre esta por padrão.

let: Cria uma gaveta que pode ter o valor substituído ao longo do tempo. Use quando souber que o dado vai mudar (como a pontuação de um jogo).

(Nota: Você talvez veja o var em códigos antigos. Ele caiu em desuso porque causava alguns comportamentos confusos no código. Evite usar).

JavaScript

const nome = "Lucas"; // Este nome não muda

let pontos = 10; // O jogo começou, o jogador tem 10 pontos e pode mudar no decorrer do jogo.

Regras de identificadores das variáveis

Aborda as regras gramaticais para dar nome às coisas no JavaScript.Em programação, um Identificador é simplesmente o nome que você escolhe para suas variáveis, constantes, funções ou classes. O JavaScript possui regras rígidas sobre o que é permitido e o que vai quebrar o seu código. Exemplos práticos:

Podem começar com letra, $ ou _:
O primeiro caractere do nome da sua variável é o mais restrito. Ele só pode ser uma letra (maíuscula ou minúscula), o símbolo do cifrão ($) ou o underline (_).

JavaScript

const nome = "Lucas"; // ✅ Válido (começa com letra)

const $preco = 10.90; // ✅ Válido (muito usado em bibliotecas como jQuery ou RxJS)

const _id = "usr_123"; // ✅ Válido (comum para indicar variáveis privadas/internas)

Não podem começar com números:
Embora você possa usar números no nome da variável, eles nunca podem vir na primeira posição.

JavaScript

const 1ano = 2026; // ❌ Erro de Sintaxe! O JavaScript se confunde achando que é um número puro.

const ano1 = 2026; // ✅ Válido! O número no final ou meio é totalmente permitido.

É possível usar letras ou números:
Depois do primeiro caractere, você está livre para misturar letras (de A a Z) e números (de 0 a 9).

JavaScript

const apartamento202 = true; // ✅ Perfeito

É possível usar acentos e símbolos:
O JavaScript moderno aceita caracteres Unicode, o que significa que tecnicamente você pode usar acentos. Porém, há uma pegadinha de boa prática aqui!

JavaScript

const preço = 50; // ✅ Funciona no JavaScript, mas...

const calça = "Jeans"; // ✅ Também funciona…

⚠️ Alerta de Boa Prática: Embora o JavaScript permita acentos, a comunidade global de programação evita ao máximo. Usar acentos ou o ç pode gerar problemas de codificação quando seu código for rodar em servidores internacionais ou quando você trabalhar com programadores estrangeiros. Prefira preco e calca.

Não podem conter espaços
Uma variável precisa ser uma palavra única e contínua. O computador não entende que duas palavras separadas por espaço pertencem ao mesmo nome.

JavaScript

const nome usuario = "Mariana"; // ❌ Erro de Sintaxe! O espaço quebra o código.

Para resolver isso, a comunidade JavaScript usa um padrão chamado camelCase (onde a primeira palavra é minúscula e as seguintes começam com letra maiúscula, parecendo as corcovas de um camelo):

JavaScript

const nomeUsuario = "Mariana"; // ✅ Perfeito! Padrão camelCase.

Não podem ser palavras reservadas
O JavaScript tem uma lista de palavras que pertencem à própria linguagem (como comandos e instruções). Você não pode usar essas palavras como nome das suas variáveis, senão o motor do navegador entra em curto-circuito.

JavaScript

const let = 10; // ❌ Erro! "let" é uma palavra reservada para criar variáveis.

const function = 5; // ❌ Erro! "function" é reservada para criar funções.

const const = "Oi"; // ❌ Erro! "const" é reservada.

Tipos de Dados: O que vai dentro das variaveis

number (Número): é Qualquer valor numérico, seja positivo, negativo, inteiro ou decimal. Diferente de outras linguagens que separam números inteiros de números com vírgula, o JavaScript coloca tudo na mesma caixa chamada number.Exemplo:

JavaScript

const idade = 26; // Inteiro

const preco = 99.90; // Decimal (sempre use ponto, não vírgula)

const temperatura = -5; // Negativo

Infinity (Infinito): Ele aparece quando você faz uma operação matemática cujo resultado é maior do que o maior número que o JavaScript consegue processar, ou em divisões por zero.Este é um valor numérico especial que representa, literalmente, o infinito matemático. Exemplo:

JavaScript

const resultado = 10 / 0;

console.log(resultado); // Vai exibir: Infinity

NaN (Not a Number / Não é um Número) Apesar do nome significar "Não é um Número", o tipo dele para o JavaScript ainda é considerado um number especial!

• O que é: Ele serve para indicar que uma operação matemática falhou ou deu um resultado inválido/impossível. • Exemplo: Se você tentar multiplicar um texto por um número: JavaScript const resultadoInvalido = "Olá" * 5; console.log(resultadoInvalido); // Vai exibir: NaN 4. string (Texto) Como vimos antes, serve para qualquer cadeia de caracteres (letras, palavras, frases, códigos). • O que é: Dados textuais que precisam estar envolvidos por aspas simples ('), duplas (") ou crases (` `). • Exemplo: JavaScript const nome = 'Ana'; const frase = "Estou aprendendo JS"; 5. boolean (Booleano) O tipo de tomada do JavaScript: ou está ligado, ou está desligado. • O que é: Um tipo de dado que só aceita dois valores possíveis: true (verdadeiro) ou false (falso). • Exemplo: JavaScript const estaChovendo = false; const maiorDeIdade = true; 6. null (Nulo) O null representa a ausência intencional de um valor. • O que é: Você usa o null quando quer dizer explicitamente para o computador: "Esta variável existe, mas ela está vazia de propósito neste momento". • Exemplo: Um campo de "segundo nome" em um formulário que o usuário não preencheu. JavaScript let segundoNome = null; // O desenvolvedor definiu que está vazio 7. undefined (Indefinido) O undefined significa que algo foi criado, mas nunca recebeu nenhum conteúdo. • O que é: Ao contrário do null (que é intencional), o undefined é o padrão do JavaScript para variáveis que foram declaradas com let, mas não receberam nenhum valor inicial. • Exemplo: JavaScript let minhaVariavel; console.log(minhaVariavel); // Vai exibir: undefined (porque não ganhou valor) 8. object (Objeto) Aqui entramos nos tipos estruturais. Um objeto permite agrupar várias informações em um único lugar usando pares de chave: valor. • O que é: Uma coleção de dados relacionados. É como uma ficha cadastral de alguma coisa do mundo real. • Exemplo: JavaScript const carro = { marca: "Toyota", ano: 2024, ligado: true }; 9. Array (Lista) O Array é, na verdade, um tipo especial de Objeto, mas que funciona como uma lista ordenada. • O que é: Uma caixa onde você coloca vários dados em fila, separados por vírgulas e envolvidos por colchetes [ ]. • Exemplo: JavaScript const frutas = ["Maçã", "Banana", "Morango"]; const numerosSorteados = [12, 45, 23, 8]; 10. function (Função) No JavaScript, as funções também são tratadas como um tipo de dado. Elas são blocos de código que realizam uma ação quando são chamadas. • O que é: Um "verbo" ou uma "máquina" que processa dados e te devolve uma resposta. Você pode guardá-las dentro de variáveis se quiser. • Exemplo: JavaScript function saudar() { return "Olá, mundo!"; }
